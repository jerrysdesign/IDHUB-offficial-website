基于开放原则的去中心化

数字身份管理和应用平台

IDHub 是建立在开放原则之上，基于区块链技术的去中心化数字身份应用平台，具备良好的技术兼容性与功能拓展性。作为进入数字社会的入口，IDHub通过区块链技术，对个人身份的有效性、真实性、唯一性进行合理验证，力求将身份控制权由第三方信息服务机构重新收回到个人手中，为用户塑造完整、可信的自主身份，构建以用户为主导的数字身份管理和应用平台，以及安全、自主、可信的身份管理机制，最终实现以数字身份链接一切 (社会服务、数字资产、数字生活等) 的愿景。

IDHub 支持区块链层面的价值转移与应用接口，它可以为区块链提供最基本的身份支持。基于智能合约，IDHub 为用户提供区块链身份的自主性和扩展功能。
代理合约
为用户分配一个区块链上的数字 ID，数字 ID 具有区块链所有的交易操作功能。
控制合约
保证用户对数字 ID 的完全自主性，实现复杂的区块链操作。
注册合约
实现数字 ID 和身份属性的相关性，保障数字身份的价值及区块链所提供的公正性和安全性。
IDHub 关键技术
Solidity
Solidity 是面向智能合约、语法类似 JavaScript 的高级编程语言。基于智能合约的数字身份可以采取多种形式：个人、设备、实体或机构。用例广泛，如声明、动作或交易进行数字签名和验证。由于可与区块链互动，亦可控制数字资产如数字货币或代币化的资产。
Merkle Tree
Merkle 树是一种每个叶节点都标有数据块，而每个非叶节点用其子节点标记的加密散列算法进行标记的树，通过提供适当的散列来实现最低限度的信息披露，从而只公开所需要的数据。这样可以帮助身份系统保持隐私，并减轻信息披露的潜在副作用。
OpenPDS
MIT 长期致力于开发 OpenPDS，它可以让用户存储个人数据，而用户以外的所有应用或服务只能获取有限或所需的部分数据。OpenPDS 为用户隐私及使用体验提供了一个双赢的解决方案，并且能在数字身份丰富的应用场景中扮演 Oracle 的角色。Oracle 是区块链上自动取得数据的一个可信任节点。
Kademlia
Kademlia 是一个点对点分布式哈希表，哈希表记录了不同哈希值进行异或运算的结果 (XOR) 。由于是基于异或运算的新型度量拓扑，Kademlia 具有可靠的稳定性及性能，且具有延迟最小化路由、无延迟故障恢复和对称单向拓扑等优势和特点。
JSON Web Token
身份属性的有效性通过由 JSON Web Token (JWT) 实现的认证来保证。JWT 是一种开放标准 (RFC 7519)，定义了一种紧凑且独立的方式，以在各方之间以 JSON 对象的形式安全传递信息。该信息可以通过数字签名验证和授信，签名用于验证JWT的发行者，确保该消息在传输过程中保持完整。
Identity Graph
身份图是身份之间行为的一种表示。在图中，节点就是身份，而边界就是行为。行为包括以好友身份添加、认可、评论和交易等。在区块链上，我们将这些行为转变为交易。IDHub系统中身份的声明和证明存储在 Merkle 树中，与其他身份的交互则以身份图的形式存储。
